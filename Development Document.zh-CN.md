## **1. 项目概述**

**Code-Repository** 是一个面向开发者和普通用户的在线代码与文件托管平台，类似于轻量级的 Git 仓库管理系统。平台支持用户通过邮箱注册登录，并基于 JWT 进行安全的身份认证与会话管理。

核心功能围绕仓库、文件、预览和代码质量四大模块展开：

- **仓库管理**：用户可以创建公开或私有仓库，管理分支、查看提交历史，并设置仓库权限与协作者。

- **文件管理**：支持通过拖拽、文件选择器等方式上传文件和文件夹，大文件采用分块上传并支持断点续传。用户可在仓库内进行目录/文件的创建、重命名、移动和删除操作，支持单文件下载或批量打包为 ZIP 下载。

- **在线预览**：提供丰富的预览能力，包括语法高亮的文本/代码预览、图片缩放预览、PDF 分页预览，以及音视频播放和 Office 文档预览（转换为 PDF/HTML）。同时左侧目录树支持折叠/展开，方便浏览仓库结构。

- **代码质量**：平台内置与 SonarQube 的深度集成，用户可以一键触发代码质量分析，查看质量门禁（通过/失败）、Bug 数量、漏洞、代码异味等指标。分析结果会直接展示在仓库界面中，并支持在合并请求时自动检测代码质量，作为合并的前置条件。

平台采用前后端分离架构，后端提供 RESTful API，前端实现响应式界面。通过合理的设计，旨在为个人开发者、团队协作及小型企业提供一套易用、高效、可扩展的代码与文件管理解决方案，并借助 SonarQube 帮助用户持续提升代码质量。

| | 个人资料管理 | 更新头像、昵称等 | 中 |
| | 仓库权限 | 设置仓库可见性（公开/私有）并管理协作者 | 高 |
| **仓库** | 创建仓库 | 使用名称、描述、可见性创建仓库 | 高 |
| | 仓库列表 | 列出拥有的、贡献的、标星的仓库，支持分页和搜索 | 高 |

| | 目录管理 | 在仓库内创建、重命名、删除目录 | 高 |
| | 文件管理 | 重命名、移动、删除文件 | 高 |

| | 目录树 | 在左侧显示仓库文件夹结构，支持折叠/展开 | 高 |
| | 差异比较 | 显示文本文件两个版本之间的差异 | 低（可选） |

| | 手动触发代码分析 | 用户点击按钮，通过 API 或 CI 触发 SonarQube 扫描 | 高 |
| | 自动触发分析 | 推送代码或创建合并请求时自动触发扫描 | 高 |

**Git 管理**

| **功能模块** | **功能点** | **详细描述** | **优先级** |
|:---:|---|---|:---:|
| **底层管理** | 裸仓库 (Bare Repo) | 创建仓库时在服务器后端初始化一个不含工作区的裸仓库，统一管理 Git 对象 | 高 |
| | 存储配额监控 | 实时计算 `.git` 目录大小，防止单个仓库无限膨胀导致磁盘耗尽 | 中 |
| **分支管理** | 灵活分支操作 | 支持基于特定 Commit、Tag 或现有分支创建新分支，并支持分支重命名 | 高 |
| | 默认分支策略 | 允许用户自定义默认分支（如 `main`）；在删除分支时禁止删除默认分支 | 中 |
| | **分支保护 (Protected Branches)** | 允许设置保护规则：禁止强制推送 (Force Push) 或限制特定人员才有权推送代码 | 高 |
| **版本追溯** | Commit 详情查看 | 点击提交哈希可查看该次提交的所有文件变更、作者、日期及详细 Message | 高 |
| | **差异对比 (Diff)** | 实现行级别的差异展示（Side-by-side 或 Inline 视图），支持文本高亮 | 高 |
| | 操作审计 (Audit Log) | 记录所有通过 Git 协议或网页端进行的 Push、Delete、Merge 等敏感行为 | 中 |
| **工作流支持** | 合并请求 (MR/PR) | 支持从源分支向目标分支发起合并申请，自动计算是否存在冲突 | 高 |
| | **代码质量准入** | **强制性要求**：Merge 前必须通过 SonarQube 设置的 Quality Gate，否则锁定合并按钮 | 高 |
| | Web IDE 在线编辑 | 支持在浏览器直接修改文件代码并创建一个新的 Commit 提交到指定分支 | 中 |
| **外部集成** | Webhooks | 当仓库发生 `push`、`merge` 或 `tag_push` 时，向配置的外部服务器发送 JSON 格式的通知 | 中 |
| | 部署密钥 (Deploy Keys) | 允许用户配置只读或读写权限的 SSH/HTTP Key，用于自动化脚本拉取代码 | 低 |

### **2_2. 界面需求**

前端界面需遵循“现代化、响应式、高可用”的设计原则，确保在不同分辨率的屏幕下均有良好的交互体验。以下是核心页面的界面需求：

|        **页面/视图**         | **关键界面元素**                                             | **交互描述**                                                 | **优先级** |
| :--------------------------: | ------------------------------------------------------------ | ------------------------------------------------------------ | :--------: |
|       **登录/注册页**        | 表单、验证提示、切换按钮                                     | 简洁的居中卡片式设计；支持回车快速提交；表单级与字段级实时错误校验。 |     高     |
|  **控制台大厅 (Dashboard)**  | 仓库列表卡片、搜索/过滤栏、新建按钮、动态时间线              | 展示用户拥有的仓库概览；支持按名称快速过滤；侧边栏展示最近的协作者动态或系统通知。 |     高     |
|    **新建仓库模态框/页**     | 仓库名称输入框、可见性单选框、描述文本域                     | 实时校验仓库名是否重复；明确标识“公开”和“私有”的图标和说明。 |     高     |
|   **仓库主页 (代码浏览)**    | 侧边栏目录树、文件列表、Breadcrumb导航、Readme 渲染区、最新 Commit 栏 | 左侧目录树支持折叠/展开；右侧主体显示当前目录内容；底部自动渲染 README.md 文件内容。 |     高     |
|     **文件上传交互组件**     | 拖拽释放区、进度条列表、状态图标（排队/上传中/完成/失败）    | 全局或局部拖拽遮罩；大文件上传时展示实时进度条、百分比、已上传大小和预计剩余时间；支持取消上传。 |     高     |
|       **文件预览视图**       | 顶部操作栏（下载/删除/全屏）、主体预览区                     | **代码/文本**：带有行号和语法高亮；**图片**：支持拖拽拖动和滚轮缩放；**PDF**：自带分页控件；**音视频**：标准播放器UI。 |     高     |
| **代码质量面板 (SonarQube)** | 环形图/进度条、状态徽章（Pass/Fail）、指标统计卡片           | 仓库首页展示精简版质量门禁徽章；点击后进入详情页，通过图表直观展示 Bug、异味、覆盖率等指标趋势。 |     中     |
|     **分支与提交历史页**     | 分支下拉切换器、时间轴列表、Hash 标签                        | 列表按时间倒序排列 Commit；清晰展示作者头像、提交信息、时间戳及对应的 Hash 值。 |     高     |
|     **合并请求 (MR) 页**     | Diff 视图（Side-by-side / Inline）、评论区、合并控制区       | 差异代码通过红/绿背景色高亮增加/删除的行；底部合并按钮根据 SonarQube 质量门禁状态动态启用/禁用（锁定）。 |     高     |
|        **仓库设置页**        | 选项卡侧边栏、危险操作区（红框标注）                         | 按“基础信息”、“协作者管理”、“分支保护”、“高级（删除仓库）”分页签展示，防止误操作。 |     中     |

## **3. 概要设计**

### **3_1. 目录架构**

```bash
Code-Repository/
├── backend/
    ├── cmd/
    │   └── server/
    │       └── main.go                # 程序入口：初始化配置、数据库连接及启动 Gin 服务
    ├── configs/
    │   ├── config.yaml                # 配置文件（数据库、Redis、MinIO、SonarQube 凭据）
    │   └── sonar-scanner.properties   # SonarQube 扫描器基础配置
    ├── internal/                      # 私有应用程序代码，防止被外部项目导入
    │   ├── api/                       # 接口层 (Gin Handlers)
    │   │   ├── v1/                    # API 版本控制
    │   │   └── middleware/            # 中间件（JWT 验证、权限检查）
    │   ├── service/                   # 业务逻辑层 (Business Logic)
    │   ├── repository/                # 数据访问层 (DAO)
    │   │   ├── db/                    # PostgreSQL 操作 (使用 GORM 或 pgx)
    │   │   ├── cache/                 # Redis 操作 (JWT 黑名单、上传状态)
    │   │   └── storage/               # MinIO 操作 (S3 协议对接)
    │   ├── model/                     # 数据模型
    │   │   ├── entity/                # 对应数据库表的结构体 (User, Repo, File)
    │   │   └── dto/                   # 请求/响应的 Data Transfer Objects
    │   └── worker/                    # 异步任务处理 (Asynq)
    │       ├── tasks.go               # 定义任务类型（文件合并、质量分析）
    │       └── processors.go          # 任务的具体执行逻辑
    ├── pkg/                           # 可复用的工具库
    │   ├── hash/                      # 哈希计算工具 (SHA-256)
    │   └── utils/                     # 通用工具（时间格式化、JSON 封装）
    ├── go.mod                         # Go 依赖管理
    ├── go.sum
    └── .gitignore
└── deployments/                   # 部署相关配置
    ├── docker-compose.yml         # 定义 DB, Redis, MinIO, SonarQube 服务栈
    └── Dockerfile                 # 后端多阶段构建 Dockerfile
```
